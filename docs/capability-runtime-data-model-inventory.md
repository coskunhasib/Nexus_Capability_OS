# Capability Runtime Data Model Inventory

This document completes item 33 at the design-contract level.

It inventories the current repository surfaces that shape the embedded Nexus Capability Runtime. It does not claim to know the final Nexus internals. It records what exists now, how each surface should be treated, and what remains unknown.

## Classification labels

```text
canonical — likely source-of-truth runtime field
adapter-derived — generated by adapter / bridge / worker layer
runtime-derived — generated during runtime execution
ui-only — display/export helper
fixture-only — sample/mock/trial-only
design-contract — planned contract, not implemented as code yet
unknown — cannot be trusted as final yet
```

## Current gaps

```text
skill package schema is not yet standardized
tool permission classes are not yet standardized
agent/sub-agent ownership and delegation are not encoded
memory is packet-oriented, not notes-first yet
context selection is not active-note based yet
evaluation observations are not first-class data
runtime loop is not explicitly modeled
artifact lifecycle and workspace boundary need tighter policy
```

## Contract direction

The next design items should move the model from:

```text
TaskPacket + adapter response + UI preview
```

toward:

```text
Skill package
+ Tool permission
+ Agent/sub-agent delegation
+ Memory notes
+ Active context bundle
+ Evaluation observation
+ Runtime loop cycle
```

## Item 33 completion

Item 33 is complete when this inventory is treated as the starting point for items 34-40.

It does not finalize schemas. It prevents hidden assumptions from becoming code.
